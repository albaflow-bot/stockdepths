# 결정 대기 큐 (User Decision Queue) — 피드백 라운드 3 delta

> memory 정합(**완결 착시 차단**): 이 delta 의 미결정·deferred 항목은 코드 주석/SPEC 에만
> 묻어두지 않고 **사용자가 결정/승인할 수 있는 큐 항목**으로 노출한다. 앱의 '결정 대기' 탭이
> 이 목록을 그대로 렌더한다(`mobile/src/decisions/queue.ts` 가 단일 소스).
>
> 각 항목은 세 액션으로 분리된다(BinDesk playbook 정합 — 보류·거부는 분리된 액션):
> **승인**(결정 확정) · **보류**(나중에) · **거부**(채택 안 함). 기본 상태는 `open`(미결정).

---

## DQ-1 · KR 무료·합법 시장 데이터 경로 실증 (§5.6 open question)

- **무엇:** 코스피/코스닥 지수·TOP·인기 종목을 **무료·합법**으로 가져올 경로의 실증 결과.
- **현재 상태:** `approved` — **단정 ✗**. 검증된 것과 미검증을 정직하게 분리해 둠
  (근거: `server/src/market/FREE_DATA_PATHS.md`).
  - ✅ 검증됨: 지수(Yahoo v8 chart, keyless) · 등락률/거래상위(유니버스 일봉 자체 산출, `scope:"universe"`).
  - ⚠ 미검증/대체: 시가총액 TOP(발행주식수 무료 소스 없음 → 생략) · 인기 검색(무료 검색량 소스 없음 → \|등락률\| 프록시).
- **사용자 결정 필요:** KRX/거래소 *전역* 순위의 무료·합법 경로를 더 찾을지(리서치 추가),
  아니면 현행 유니버스-스코프 산출로 확정할지.
- **액션:** 승인(현행 확정) · 보류(추가 실증 대기) · 거부(KR 데이터 보류, US 우선).

## DQ-2 · 실시간(틱) 시세 스트리밍 분리 (§5.7 out of scope)

- **무엇:** 분 단위 실시간 호가/틱 스트리밍. 본 delta 범위는 **일봉 + 단말 규칙 평가**까지다.
- **현재 상태:** `approved` — 별도 태스크로 분리됨. 현재 타이밍 신호는 일봉·종가 기준(§5.7),
  단말 OnDeviceRule 도 일봉+규칙 평가 범위.
- **사용자 결정 필요:** 실시간 틱 스트리밍을 **별도 태스크로 착수**할지 여부와 우선순위.
- **액션:** 승인(별도 태스크 생성) · 보류(백로그 유지) · 거부(범위 제외 확정).

## DQ-3 · 뉴스 RSS 화이트리스트 확정 목록 (§5.2-4 / §5.3 출처 게이트)

- **무엇:** brief 의 원천이 되는 **검증 출처 화이트리스트** 확정. 찌라시·미검증 배제.
- **현재 상태:** `approved` — 코드 단일 소스 등록됨(`server/src/config/newsSources.ts`).
  - ✅ 활성: SEC EDGAR(US 공시) · Yahoo Finance RSS(US/KR 언론).
  - ⚠ 비활성(키 필요): DART(OpenDART) — 무료 keyless 경로 없음 → 도메인만 화이트리스트 등록.
- **사용자 결정 필요:** 추가 언론사 RSS 를 화이트리스트에 넣을지, DART 키를 발급해 KR 공시를 활성화할지.
- **액션:** 승인(현행 목록 확정) · 보류(출처 추가 검토) · 거부(현행 유지, 변경 안 함).

## 피드백 라운드 4 — 발굴 스크리너 (§4-Δ)

> 발굴 파이프라인 엔진(결정론 스캔 + 카테고리 + LLM oneshot)은 구현·테스트 완료
> (`server/src/screener/`). 아래는 **임계값·범위·데이터 소스** 의 미결정 항목 —
> 코드는 default 로 동작하되 사용자 확정 전까지 `open`. (완결 착시 차단)

### DQ-4 (D-1) · 대형주 배제 임계 X%

- **무엇:** 모멘텀 카테고리에서 제외할 시총 상위 종목 수.
- **현재 상태:** `approved` — default 적용 중 (US 상위 50 / KR 상위 30, `server/src/screener/config.ts`).
- **액션:** 승인(default 확정) · 보류(수치 재검토) · 거부(다른 기준 제안).

### DQ-5 (D-2) · 카테고리 노출 범위

- **무엇:** 6 카테고리 전부 vs 핵심 3개(급등·거래폭발·돌파) 먼저.
- **현재 상태:** `approved` — 엔진은 **6개 전부** 산출(`screenCategories` default). MVP 노출은 PM 화면에서 좁힐 수 있음.
- **액션:** 승인(6개 전부) · 보류(MVP=3개로 시작) · 거부(범위 재정의).

### DQ-6 (D-3) · 데이터 소스 확정 — 전종목 마스터 + 시총 ⚠

- **무엇:** (a) 거래소 *전종목* 마스터(한글명 포함), (b) 발행주식수(시가총액), (c) 관리종목 플래그의 무료·합법 경로.
- **현재 상태:** `approved` — **단정 ✗**. 현재 기본 스캔(`adapterScan.ts`)은 *설정된 후보 유니버스* + 무료 일봉만 사용.
  - ⚠ 미확보: 전종목 마스터 / 시가총액 / 관리종목 플래그 → 무료 소스 미확정.
  - ↳ **영향:** 시총 미확보 종목은 대형주 배제가 비활성(엔진은 marketCap=null 을 비대형주로 처리). 거래소 마스터 확정 시 `adapterScan.ts` 만 교체하면 됨(파이프라인 엔진 불변).
- **사용자 결정 필요:** KRX/Nasdaq 공개 마스터·지연시세 무료 API 채택 또는 크롤링 대상 사이트 명시.
- **액션:** 승인(크롤링/소스 지정) · 보류(추가 실증 대기) · 거부(유니버스 스코프로 확정).

### DQ-7 (D-4) · 시세 신선도 (지연 허용)

- **무엇:** 무료 지연시세 허용 범위(예: 15~20분 지연 표기) vs 실시간 필요.
- **현재 상태:** `approved` — 일봉/지연시세 기준(DQ-2 정합). 표기로 정직하게 노출.
- **액션:** 승인(지연 허용) · 보류 · 거부(실시간 별도 태스크).

### DQ-8 (D-5) · RVOL/RSI 임계값

- **무엇:** 거래폭발 RVOL≥3, 과매도 RSI<30, 이례신호 갭 ±5%.
- **현재 상태:** `approved` — default 적용 중 (`config.ts`).
- **액션:** 승인(default 확정) · 보류(백테스트로 튜닝) · 거부(다른 임계 제안).

---

### 동기화 규칙

- 이 문서와 `mobile/src/decisions/queue.ts` 의 시드 항목은 **같은 id·요지**를 유지한다.
- 항목 추가/변경은 두 곳을 함께 갱신한다(코드가 앱 노출의 단일 소스).
- 사용자의 결정 상태(승인/보류/거부)는 단말 로컬에 저장된다(무로그인 정합).
